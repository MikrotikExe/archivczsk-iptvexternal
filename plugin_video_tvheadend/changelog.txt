Tvheadend plugin pre archivCZSK
Verzia 0.70.0 (26.5.2026)
================================================================

HTSP režim — prepracovaný a stabilizovaný:

  - Streamovanie v HTSP režime ide teraz PRIAMO cez TVH HTTP
    endpoint (stream/channelid/, dvrfile/) na porte 9981, rovnako
    ako v HTTP režime. HTSP (9982) sa používa už len na DÁTA
    (kanály, EPG, DVR archív, tagy). Dôvod: prehrávač exteplayer3
    nemá demuxer API ako Kodi pvr.hts, takže natívne HTSP
    streamovanie na Enigme nie je možné. Stačí mať dostupný port
    9981 pre stream (web rozhranie môže byť vypnuté).

  - Vlastný HTSP→MPEG-TS muxer a interný streaming proxy (P_htsp)
    úplne odstránené — nepoužívali sa.

  - Načítanie dát: jeden zdieľaný HTSP fetch naplní kanály, tagy,
    DVR aj EPG naraz (server ich posiela v jednom prúde). Zámok
    (lock) zabraňuje dvom paralelným fetchom súčasne — chráni RAM
    servera pred preťažením.

  - Rýchle spustenie kanála: pre streamovanie sa ťahajú len kanály
    (rýchla cesta, ~1-2s) bez čakania na celý DVR/EPG dump. Po
    reštarte GUI sa kanál spustí okamžite namiesto desiatok sekúnd.

  - Dvojfázový prefetch pri štarte: najprv rýchlo kanály, potom
    (s odstupom) plný EPG+DVR pre archív. Nezdržuje prvé prehratie.

  - EPG cache: neúplný EPG (keď server nestihne poslať celý) sa
    necachuje nadlho — pri ďalšom otvorení sa dotiahne kompletný.
    Časový limit fetchu zvýšený pre veľké archívy (8000+ nahrávok).

  - DVR archív sa obnovuje s rozumným TTL (10 min) — nové nahrávky
    pribudnú automaticky pri ďalšom otvorení.

  - Opravená kategorizácia archívu (Filmy/Seriály/Šport/...) aj
    poradie kategórií v automaticky generovanom userbouquete.

  - Interný audit: odstránených ~700 riadkov mŕtveho kódu,
    nepoužívané importy a premenné.

Tvheadend plugin pre archivCZSK
Verzia 0.62.0 (25.5.2026)
================================================================

HTSP režim pripojenia (nové):

  - Pridaná podpora pripojenia cez natívny HTSP protokol (port
    9982) ako alternatíva k HTTP API (9981). Prepínač "Connection
    mode" v nastaveniach (HTTP API / HTSP).

  - HTSP režim pokrýva kompletnú funkcionalitu: zoznam kanálov,
    EPG, DVR archív, živé vysielanie aj picony. Bouquet export a
    EPG injekcia fungujú rovnako ako v HTTP režime.

  - DVR archív cez HTSP: nahrávky sa čítajú metódou fileOpen /
    fileRead (/dvrfile/), čím sa získa originálny MPEG-TS súbor so
    všetkými pôvodnými stopami (viac audio vrátane AC3 5.1,
    teletext, DVB titulky).

  - HTSP autentifikácia používa SHA1 digest z výzvy servera, takže
    nie je dotknutá nastavením typu HTTP digest hashu.

  - Pri zlyhaní HTSP modulu doplnok automaticky funguje v HTTP
    režime (žiadny dopad na existujúce inštalácie).

Tvheadend plugin pre archivCZSK
Verzia 0.61.0 (23.5.2026)
================================================================

Pripojenie / autentifikácia:

  - Pridaná podpora Digest autentifikácie s hashom SHA-256 a
    SHA-512/256 (predtým len MD5). Plugin teraz číta typ hashu
    priamo z odpovede servera (WWW-Authenticate) a počíta digest
    správnym algoritmom podľa RFC 7616.

  - Rieši problém keď sa plugin nedokázal pripojiť k serveru,
    ktorý mal v nastaveniach "Digest hash type" nastavený na
    SHA-256 alebo SHA-512/256, hoci meno a heslo boli správne.
    (Konfigurácia servera: HTTP Server Settings -> Digest hash
    type.) Funguje teraz so všetkými troma typmi hashu bez
    nutnosti meniť nastavenia servera na MD5.


Tvheadend plugin pre archivCZSK
Verzia 0.60.0 (22.5.2026)
================================================================

Klient pre Tvheadend server na sledovanie živej TV, rádia
a prehrávanie DVR archívu s vyhľadávaním.


ČO PLUGIN PONÚKA
----------------------------------------------------------------

Živé TV a rádio:
  - Streamovanie kanálov priamo z TVH servera cez HTTP/HTTPS
  - Podpora HTTP Digest a Basic autentifikácie s auto-detekciou
  - Ticket URL pre HLS-AES enkryptované streamy
  - Výber stream profilu (pass, mpegts, matroska, htsp, webtv)
  - Výber prehrávača: Default, gstplayer, exteplayer3

DVR archív:
  - Prehliadanie všetkých nahrávok na TVH serveri
  - Pokračovanie prehrávania od poslednej pozície
  - Hviezdičkový marker (★) pri pozretých nahrávkach (nad 80 %)
  - Automatická kategorizácia (filmy, seriály, šport, dokumenty,
    spravodajstvo, hudba, detské, archív)
  - Voliteľné vyhľadávanie titulov cez IMDb (opt-in)
  - Vyhľadávanie cez všetky nahrávky

Vytvorenie userbouquet:
  - Automatický export TVH kanálov do Enigma2 userbouquet
  - Separátne bouquets pre TV a rádio s vlastnými názvami
  - Konfigurovateľný interval automatickej obnovy
  - Pridávanie názvov kategórií ako separátory

Sťahovanie picons:
  - Automatické sťahovanie picons z TVH servera
  - SRP formát (Service Reference Pattern)
  - Cache pre 404 (broken icons sa neretryjú zbytočne)

Vkladanie EPG do Enigma2:
  - Priame vkladanie EPG do Enigma2 (bez epgimport pluginu)
  - Automatický refresh každé 4 hodiny + pri zmene nastavení
  - Konfigurovateľný počet dní EPG histórie (1-14)

Diagnostika:
  - Test TVH prihlásenia a pripojenia
  - Zobrazenie ciest (stamps, cache, picon directory)
  - Manuálne spustenie obnovy bouquetu a EPG
  - Vyčistenie 404 picon cache

Watchdog:
  - Automatické sledovanie offline → online prechodu TVH servera
  - Pri detekcii online stavu spustí obnovu bouquetu a picons
  - Netreba manuálne otvoriť plugin po reštarte TVH servera


NOVINKY VO VERZII 0.60.0
----------------------------------------------------------------

Táto verzia prináša kompletné vyriešenie zobrazovania piconov
(loga kanálov) pri Tvheadend kanáloch a niekoľko súvisiacich
vylepšení obnovy bouquetu.

Picony:

  - Picony sa teraz správne zobrazujú pri Tvheadend kanáloch —
    rovnako ako pri IPTV M3U doplnku. Predchádzajúce verzie
    ukladali picon súbory pod nesprávnym menom (nesúlad typu
    služby aj identifikátora kanálu), takže Enigma2 ich pri
    kanáloch nenašla, hoci boli stiahnuté na disku. Picony sa
    teraz ukladajú pod menom ktoré presne zodpovedá kanálu
    v zozname.

  - Picony naskočia ihneď po obnovení bouquetu, bez nutnosti
    reštartu grafického rozhrania. Akcie v nastaveniach vykonajú
    rovnaké obnovenie service listu ako pri reštarte.

  - Pri použití kategórií (kanál môže patriť do viacerých
    kategórií naraz) sa logo priradí ku správnemu kanálu
    spoľahlivo.

  - Nová akcia v nastaveniach: "Vynútiť opätovné stiahnutie
    všetkých piconov" — zmaže staré picony Tvheadend kanálov
    a stiahne ich nanovo. Užitočné keď sa logo kanálu na serveri
    zmení. Maže len picony Tvheadend kanálov; ostatné (napr.
    IPTV M3U) ostávajú nedotknuté.

Obnova bouquetu:

  - Akcia "Obnoviť TVH bouquet + XML EPG" je teraz plná obnova:
    pred generovaním zmaže starý zoznam kanálov z cache aj staré
    bouquet súbory a natiahne čerstvé dáta zo servera. Vyrieši to
    prípad keď sa kanály na serveri zmenili (pribudol/ubudol
    kanál, zmena poradia).

  - Opravené zdvojené kanály v zozname. Keď prebehli dve obnovenia
    súčasne (napr. automatické + manuálne, alebo rýchle
    dvojkliknutie), zoznam kanálov sa mohol zdvojiť. Načítanie
    kanálov je teraz odolné voči súbehu a obsahuje ochranu proti
    duplikátom — každý kanál sa v zozname objaví práve raz.

  - Manuálne akcie obnovy a sťahovania piconov bežia na pozadí,
    takže menu počas operácie nezamrzne. Priebeh je viditeľný
    v archivCZSK logu.
